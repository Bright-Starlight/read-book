# 工作流参考

此参考文件用于整本书的 CLI 深读流程。

## 推荐执行顺序

1. 识别文件类型（PDF 或 EPUB）。
2. 提取文本与结构标记。
3. 提取或恢复目录。
4. 先按章切分，再按节切分长章节。
5. 检查视觉材料：图片、图表、表格、地图、时间线。
6. 为每章先做节级笔记，再合成为 `chapters/chNN_章节精读.md`。
7. 完成全书级文件。
8. 统一术语、编号、章节命名和中文风格。
9. 做一次“细节回收”检查，确认没有漏掉关键定义、图像信息与限定条件。

## 结构恢复优先级

1. EPUB 自带目录或 PDF 书签
2. 正文目录页
3. 标题编号模式（如“第 3 章”“III”“3.1”）
4. 版式、字号、留白模式
5. 语义转折（最后手段）

## 优先保留的内容

- 章标题、节标题
- 定义性引文
- 表格与分类框架
- 图注、表注
- 会改变解释的脚注
- 关键日期、数据、地点、人物、研究名称
- 说明概念差异或机制关系的图

## 可以压缩的内容

- 出版信息
- 鸣谢
- 装饰性题辞
- 不影响理解的重复页眉页脚
- 纯装饰性图片

## 发现精读过浅的信号

- 只剩观点，没有推理过程
- 只说“作者认为”，却没解释“作者怎么证明”
- 漏掉关键定义或概念差异
- 案例只点名，不说明作用
- 图表存在却完全不处理
- 没写范围、前提或例外

## 图片与图表检查清单

每章至少快速检查一次：
- 是否有图题或图注
- 是否有表格表头
- 是否有坐标轴、图例或颜色分组
- 是否有地图、时间线、关系图
- 正文是否明确引用“见图”“如下表”“如图所示”
- 若视觉材料缺失，是否已在章节中说明这一缺口


## CLI 目录工作流补充

10. 在 `work/` 中写出 `book_structure.json`、`chapter_index.json`、`images_manifest.json`。
11. 只把成品 Markdown 放入 `output/`。
12. 对所有需要画出的结构图、机制图、时间线、复习路径图，统一使用 Mermaid 语法。
13. `05_复习问题.md` 题量默认偏多，优先保证二刷可用，而不是只给一组简短问答。

## Mermaid 使用建议

优先使用：
- `flowchart`：论证推进、章节关系、工作流
- `mindmap`：概念网络、复习结构
- `timeline`：历史书时间线
- `graph LR` / `graph TD`：机制图、变量关系图

注意：
- 图不能代替解释，图前后都要有文字说明。
- 节点命名尽量中文简洁，避免大段长句塞进节点。
