# Social Book Decomposer

当用户提供整本 PDF 或 EPUB，希望在 CLI 工作流中自动完成预处理、章节切分、逐章精读和全书级综合时，遵循本技能。


优先参考以下资源：
- 工作流说明：`references/workflow.md`
- 输出模板：`references/output-template.md`
- 书型分支：`references/book-types.md`
- CLI 命令配方：`references/cli-recipes.md`
- 初始化脚本：`scripts/run_book_pipeline.py`
- 一键执行脚本：`scripts/run.sh`
- 真实提取脚本：`scripts/extract_book.py`
- 渲染草稿脚本：`scripts/render_book.py`
- 章节骨架脚本：`scripts/scaffold_chapters.py`
- 输出校验脚本：`scripts/validate_outputs.py`

## 目标

产出的 Markdown 必须同时满足四个目标：

1. 让读者能在很短时间内掌握全书主线。
2. 让读者能回到任一章节做深度复习。
3. 保留作者是如何论证的，而不只是保留结论。
4. 尽可能保住图片、图表、表格、地图、时间线、脚注和附录中真正影响理解的内容。

默认输出语言为中文。除专有名词、原文术语、章节标题确有保留必要外，不要用英文替代中文说明。

在开始章节分析前，必须先判断本书更接近哪一种书型，并加载 [书型分支模板](references/book-types.md)。书型判断会改变章节精读与全书综合的侧重点。

## 脚本使用原则

- 本技能附带的是**最小可运行工具链**，用于初始化目录、真实提取、生成索引、渲染 Markdown 草稿、校验产物完整性。
- 不要把脚本误解为“已经自动完成高质量学术精读”。内置的 `scripts/extract_book.py` 和 `scripts/render_book.py` 负责生成可靠的第一版工作草稿；若你有更强的提取器或模型调用脚本，可以覆盖。
- 若用户只想看方法论，可不运行脚本，直接按模板执行。
- 若用户在 CLI 中实际跑流程，优先先运行一键入口：

```bash
bash scripts/run.sh --book-dir ./my-book
# 默认会调用 scripts/extract_book.py 回填 work/extracted_text.md 等文件
```

若只想初始化骨架，再单独运行：

```bash
python scripts/run_book_pipeline.py --book-dir ./my-book
```

在章节与全书级文件完成后，也可单独运行：

```bash
python scripts/validate_outputs.py --book-dir ./my-book
```

## Claude Code CLI 一键执行约定

在 Claude Code CLI 中，优先把本技能当作“可接钩子的总控脚本”使用，而不是手工逐步敲命令。

推荐入口：

```bash
bash scripts/run.sh --book-dir ./my-book
# 默认会调用 scripts/extract_book.py 回填 work/extracted_text.md 等文件
```

若你已经有自己的提取器、目录恢复器、渲染器，可直接挂到一键入口上：

```bash
bash scripts/run.sh --book-dir ./my-book \
  --extract-cmd 'python tools/extract.py --book-dir "$BOOK_DIR"' \
  --index-cmd 'python tools/build_index.py --book-dir "$BOOK_DIR"' \
  --render-cmd 'python tools/render_notes.py --book-dir "$BOOK_DIR"'
```

默认阶段为：
1. 初始化目录与骨架文件
2. 默认执行内置基础提取；如有需要可用自定义命令覆盖
3. 如需覆盖或修正目录，可再执行自定义章节索引命令
4. 根据 `chapter_index.json` 生成逐章精读骨架
5. 默认执行内置渲染草稿；如有需要可用自定义命令覆盖
6. 运行最小产物校验

其中 `extract-cmd`、`index-cmd`、`render-cmd` 都是可替换钩子。默认情况下，一键脚本会先运行内置基础提取器，再运行内置 Markdown 渲染草稿器；若你已有更强的 PDF/EPUB 解析器、章节识别器、模型调用脚本，可以无缝替换而不改动目录规范。

## 总体流程

1. **预处理源文件**
   - EPUB 优先使用自带目录与 spine 顺序。
   - PDF 先尽量提取带有标题层级的干净文本，再识别目录和章节边界。
   - 清理页眉、页脚、页码、重复标题、扫描噪声、断裂段落、重复 OCR 片段。
   - 保留会影响解释的前言、导论、序言、附录、脚注与图注；其余前置出版信息可压缩。

2. **恢复全书结构**
   - 优先使用 EPUB 目录、PDF 书签、正文目录页、编号模式、版式层级来恢复章节。
   - 先按“章”切分；章节过长时，再按“节/小节”切分。
   - 维护章节编号、原标题、别名、页码范围或定位信息之间的映射。

3. **逐章精读**
   - 章节是主分析单元，不要只按 token 窗口机械摘要。
   - 长章节先形成节级笔记，再合成为章节精读。
   - 章节精读必须保留：核心问题、主张、论证链、概念定义、证据材料、前提条件、反例或例外、容易忽略的细节、图像信息。

4. **书型判断与分支加载**
   - 在正式写作前，先判断本书属于：理论书、历史书、社会学/政治学书、访谈/田野材料型作品中的哪一类。
   - 若为混合型，指定一个主分支，再少量吸收次分支的关注点。
   - 将书型判断和依据写入 `00_readme.md`。

5. **全书综合**
   - 所有章节完成后，再生成全书结构图、核心论证、关键概念表、易误读点、复习问题。
   - 对跨章节反复出现的概念，标明含义是否发生变化。
   - 根据书型额外生成相应的全书级附加模块。

6. **质量复核**
   - 检查是否存在“只剩结论、不见推理”的浅摘要。
   - 检查是否遗漏关键图片、表格、图注、定义、限定条件。
   - 检查中文输出是否自然、准确，术语是否统一。

## 预处理要求

### EPUB

- 优先读取目录、spine、章节 HTML 结构。
- 尽量保留标题层级、引用、列表、脚注、强调、图注。
- 若一个章节被分散在多个 HTML 文件中，应先合并再分析。
- 若目录与正文标题不一致，记录差异并优先服从正文可验证结构。

### PDF

- 先尽量恢复标题层级，再做语义分析。
- 通过书签、目录页、标题字号、空白间距、章节编号模式、页面跳转来识别章节边界。
- OCR 质量差时，要明确说明哪些位置可能不可靠。
- 对扫描版 PDF，不要假装精确识别成功；宁可保留不确定性说明。

## 图片、图表、表格与视觉材料处理

视觉内容不能被当作噪声忽略。凡是可能影响理解的视觉材料，都必须在章节精读中显式处理。

### 必须关注的视觉对象

- 插图、照片、漫画、示意图
- 统计图、折线图、柱状图、散点图、流程图
- 地图、时间线、关系图
- 表格、对照表、分类框架
- 图注、表注、图内文字

### 处理规则

1. **先判断是否有解释价值**
   - 如果图片只是装饰，可略写为“装饰性插图，无关键分析信息”。
   - 如果图片参与论证，必须进入章节精读正文。

2. **提取图像的分析作用**
   - 说明这张图/表在作者论证中承担什么功能：举例、证明、分类、对比、历史定位、概念可视化，还是情绪强化。

3. **记录可验证内容**
   - 标题、图注、坐标轴、图例、表头、时间点、地点、人物名称、分类标签、显著趋势。
   - 不要编造看不清的数字；不确定时要写明“图中数字无法可靠辨认”。

4. **输出到章节精读**
   - 相关内容至少要出现在 `## 图像、图表与表格信息` 一节。
   - 若某张图是章节关键证据，也要在 `## 证据、案例与材料` 里再次提及。

5. **必要时单独提醒**
   - 若纯文本提取明显丢失图片信息，要在 `00_readme.md` 和对应章节文件里明确提示：该章可能需要人工回看原书图片。

6. **优先回收低成本高价值信息**
   - EPUB 优先回收：`figcaption`、`alt`、`title`、`aria-label`、附近正文对图表的提及。
   - PDF 优先回收：同页 caption 候选、页面文本中的“见图/如下表”类正文引用、文件名或对象名线索。
   - 在没有 OCR 的前提下，优先保证“图像上下文可用”，而不是假装已经识别图中全部细节。

### 图片信息不足时的降级策略

- 优先利用图注、正文引用、上下文描述还原图片作用。
- 若只能确认“有图但无法可靠读取细节”，不要猜测图像内容；写出可确认部分与缺失部分。


## CLI 工程化约定

当在 CLI 中执行时，默认遵循以下约定：

### 输入目录

- `input/`：用户放入原始书籍文件，只保留 1 本当前待处理书籍。
- 文件名尽量稳定，优先使用 `source.pdf` 或 `source.epub`。

### 中间产物目录

- `work/extracted_text.md`：预处理后得到的连续正文。
- `work/toc_raw.md`：从目录页、书签或 EPUB 导航中恢复的原始目录。
- `work/book_structure.json`：全书结构索引。
- `work/chapter_index.json`：章节级元数据索引。
- `work/images_manifest.json`：图片、图表、表格、地图、时间线的记录清单。
- `work/chapter_chunks/`：超长章节被拆成的小段 Markdown，中间分析用，不作为最终交付。

### 输出目录

- `output/`：最终交付给读者的 Markdown 文件。
- 除非用户特别要求，最终只把可读成品放在 `output/`，不要把噪声中间文件混进来。

### JSON 索引要求

#### `work/book_structure.json`

至少包含：
- `title`
- `source_type`
- `book_type`
- `structure_confidence`
- `chapters`：数组，元素至少含 `chapter_id`、`title`、`order`、`start_locator`、`end_locator`
- `front_matter`
- `appendix`
- `notes_on_extraction`

#### `work/chapter_index.json`

至少包含每章：
- `chapter_id`
- `chapter_title`
- `chapter_aliases`
- `order`
- `start_locator`
- `end_locator`
- `subsections`
- `has_visual_material`
- `visual_ids`
- `analysis_status`

#### `work/images_manifest.json`

至少包含每个视觉对象：
- `visual_id`
- `chapter_id`
- `type`
- `title_or_caption`
- `locator`
- `is_key_evidence`
- `extractable_fields`
- `confidence`
- `notes`

### 文件命名规则

- 章节文件统一使用两位或三位序号，如 `ch01_章节精读.md`。
- 若章节标题太长，只把编号放在文件名中，把全标题写在文件内一级标题。
- 图表索引中的视觉对象编号可用 `fig-ch03-02`、`tbl-ch05-01` 这类形式。

### 失败与降级

- 结构恢复不稳时，仍然生成索引文件，但必须在字段里显式写低置信度和原因。
- 图片无法可靠解析时，也要在 `images_manifest.json` 与对应章节里保留占位记录，不可静默丢弃。
- 对长章节切块后，最终章节精读仍然必须回到“整章视角”统一写作，不可把分块摘要原样拼接。


## 必须生成的 Markdown 文件

默认生成以下文件，除非用户明确要求减少：

```text
book-name/
  input/
    source.pdf | source.epub
  work/
    extracted_text.md
    toc_raw.md
    book_structure.json
    chapter_index.json
    images_manifest.json
    chapter_chunks/
      ch01_part01.md
      ch01_part02.md
      ...
  output/
    00_readme.md
    01_全书结构图.md
    02_核心论证.md
    03_关键概念.md
    04_易误读与限制.md
    05_复习问题.md
    06_图表索引.md
    chapters/
      ch01_章节精读.md
      ch02_章节精读.md
      ...
```

若用户没有指定目录，可默认把最终 Markdown 文件放在 `output/`，把中间产物放在 `work/`。

### 00_readme.md

必须包含：
- 源文件名
- 文件类型
- 书型判断
- 判断依据
- 实际启用的分支关注点
- 结构恢复置信度（高 / 中 / 低）
- 识别到的章节数量
- 预处理中的主要问题
- 图片/图表保留情况说明
- 其他文件如何配合使用的简短说明

### 01_全书结构图.md

必须包含：
- 全书核心问题
- 全书主论题或中心回答
- 逐章功能图
- 论证推进路径
- 哪几章是奠基章、推进章、收束章、案例章或方法章
- 至少 1 个使用 Mermaid 语法绘制的结构图（例如流程图、章节关系图或论证推进图）

### 02_核心论证.md

必须包含：
- 作者的中心命题
- 3 到 7 个最重要的支撑论证
- 论证之间的依赖关系
- 作者最关键的前提
- 主要证据类型
- 重要的适用边界、例外或限制

### 03_关键概念.md

对每个关键概念都要写：
- 术语
- 中文解释
- 作者如何使用它
- 它与哪些相近概念容易混淆
- 它在全书哪几章最关键
- 必要时保留原文术语

### 06_图表索引.md

必须包含：
- 按章节汇总出现过的关键图片、图表、表格、地图、时间线
- 每个视觉对象的标题或占位标题
- 它在论证中的作用
- 可确认字段与缺失字段
- 至少 1 个使用 Mermaid 语法绘制的全书图表分布图、视觉证据分布图或图像处理流程图

### 04_易误读与限制.md

必须包含：
- 最容易被过度简化的判断
- 最容易混淆的概念对
- 作者结论成立所依赖的条件
- 只是示例、不足以单独证明结论的案例
- 文本或图像提取不可靠的位置

### 05_复习问题.md

必须包含：
- 20 到 40 个复习问题，其中至少分为：基础理解、论证追踪、概念辨析、批判思考、迁移应用 五组
- 8 到 12 个必须记住的记忆锚点
- 5 到 10 个迁移应用问题
- 适合二刷时重点核对的章节提醒
- 至少 1 个用 Mermaid 语法绘制的复习路径图、概念复习图或章节回看顺序图

## 每个章节文件的固定结构

章节文件命名为 `chapters/chNN_章节精读.md`。

每个章节文件必须严格按以下顺序包含这些部分：

1. `# 章节标题`
2. `## 本章在全书中的位置`
3. `## 本章要回答的核心问题`
4. `## 本章的核心主张`
5. `## 论证链条拆解`
6. `## 关键概念与概念区分`
7. `## 证据、案例与材料`
8. `## 图像、图表与表格信息`
9. `## 前提、限制与例外`
10. `## 容易被忽略的细节`
11. `## 一分钟回看`

## 章节精读的详细写法

### 1. 本章在全书中的位置

说明这一章是：
- 开题、定义、搭框架
- 反驳既有观点
- 推进中心论证
- 提供案例或经验材料
- 收束、比较、扩展，还是回应反对意见

### 2. 本章要回答的核心问题

不要泛泛写主题，要写成真正的问题句。尽量指出：
- 作者为什么在这里提出这个问题
- 它与前一章的连接点是什么

### 3. 本章的核心主张

用 1 到 3 段说明本章最重要的判断。必须区分：
- 作者明确提出的主张
- 作者暗中依赖但未充分展开的判断

### 4. 论证链条拆解

这是最重要的部分，默认写得比其他部分更长。至少要做到：
- 按步骤拆解作者如何从前提走向结论
- 明确每一步在做什么：定义、区分、举例、反驳、归纳、比较、建模、历史解释、因果说明
- 指出论证哪里最强，哪里较弱，哪里跳步
- 若本章包含对其他学者或流派的批评，写清被批评对象、批评点、替代方案

不要把这一节压缩成几条空泛 bullet。默认使用成段中文，必要时再加小标题。

### 5. 关键概念与概念区分

对关键概念要写出：
- 定义
- 在本章中的功能
- 与相邻概念的区别
- 是否与全书其他章节的用法不同

特别关注：
- 因果 vs 相关
- 描述性判断 vs 规范性判断
- 理想类型 vs 经验事实
- 机制 vs 结果
- 条件 vs 结论

### 6. 证据、案例与材料

写清作者用了什么来支撑本章：
- 历史材料
- 田野或访谈
- 数据与统计
- 二手文献
- 比较案例
- 个人叙述或思想实验

不要只罗列，要说明每类材料支撑了哪一步论证。

### 7. 图像、图表与表格信息

必须回答：
- 本章是否存在关键视觉材料
- 它们分别说明什么
- 哪些数据、标签、图注或关系值得保留
- 它们在论证中是核心证据、辅助说明，还是仅作展示

如果无法可靠识别，也要说明无法识别的部分。

### 8. 前提、限制与例外

至少检查这些内容：
- 作者默认了哪些前提
- 结论适用于什么范围
- 哪些情况会削弱本章结论
- 作者有没有承认反例、例外、边界条件

### 9. 容易被忽略的细节

这一节不能空泛，优先保留：
- 一句关键定义
- 一个常被略过的限定语
- 一个不起眼但支撑全章的案例
- 一个脚注、图注或术语差异
- 一个决定本章逻辑方向的转折句

### 10. 一分钟回看

用一小段中文写出本章最值得二刷时回看的内容，要求能在一分钟内重读完。

## 写作硬规则

- 默认使用中文完整段落，不要只给提纲式摘要。
- 精读优先保真，不为了简洁牺牲关键细节。
- 不得把一章压缩成“主题列表”。
- 需要保留分析重量足够的案例、图表、表格与定义。
- 要明确区分“作者原意”“文本可见证据”“整理后的综合判断”。
- 出现提取不确定时，必须标注不确定性，不得伪造精确内容。
- 若某章主要是历史叙述，保留时间顺序的前提是它服务于解释。
- 若某章主要是否定性批评，必须写出被批评对象、理由与作者替代方案。
- 需要画图时，只能使用 Mermaid 语法，不要输出 ASCII 图、伪代码图或其他自创图示格式。
- Mermaid 图必须服务于理解，优先用于：章节关系、论证推进、机制图、时间线、概念网络、复习路径。
- 所有 Mermaid 图前后都要配简短中文说明，避免只有图没有解释。

## 全书综合规则

全部章节完成后，必须再做一次全书层面的交叉检查：

- 哪些概念在不同章节含义发生了变化
- 哪些章节是基础章节，哪些只是补充或延伸
- 哪些内容是全书不可压缩的核心，哪些属于背景铺垫
- 书中是否存在尚未解决的张力、矛盾或开放问题
- 图片与文字证据之间是否相互支撑

## 何时判定精读过浅

出现以下情况之一，就说明结果不合格，需要补强：

- 只有结论，没有论证步骤
- 有主题，没有概念定义
- 有概念，没有概念区分
- 有案例名称，没有说明案例如何支撑论证
- 完全漏掉图表、插图、图注
- 没写适用边界、例外或前提
- 中文虽然通顺，但读完仍看不出作者是怎么想出来的

## 提取质量较弱时

若预处理质量低：
- 明确指出不可靠位置
- 继续保留能恢复出的最好结构
- 优先保留高价值章节与核心概念
- 不要为了“完整”而编造图片、数字、章节边界或引用


## 书型分支模板

必须阅读并执行 [书型分支模板](references/book-types.md)。

不要把不同书型混成同一套空泛摘要逻辑。统一主模板只负责保证文件结构一致；真正的精读重点要由书型分支决定。

- **理论书**：优先追踪概念工程、理论对手、命题支点与推理跳步。
- **历史书**：优先追踪时间线、转折点、史料来源与历史解释。
- **社会学 / 政治学书**：优先追踪研究问题、因果机制、变量条件、比较框架与替代解释。
- **访谈 / 田野材料型作品**：优先追踪材料来源、样本边界、叙事与分析切换、方法位置与代表性问题。

如书型判断错误，后续章节精读会系统性失焦，因此这一步不能省略。


## 脚本清单

- `scripts/run.sh`：一键串起初始化、可选提取、可选索引、章节骨架生成、可选渲染与校验。
- `scripts/run_book_pipeline.py`：仅初始化目录与骨架文件。
- `scripts/scaffold_chapters.py`：根据 `work/chapter_index.json` 生成逐章精读 Markdown 骨架。
- `scripts/validate_outputs.py`：校验 `work/` 与 `output/` 的最小交付完整性。


## 内置基础提取器

新增脚本 `scripts/extract_book.py`，用于把真实提取层接到工程流中。

### EPUB
- 自动读取 `META-INF/container.xml` 与 OPF。
- 优先读取 nav 目录并回填 `work/toc_raw.md`、`work/chapter_index.json`。
- 按 spine 顺序提取正文，写入 `work/extracted_text.md`。
- 检测 HTML 中的 `<img>`，写入 `work/images_manifest.json`。

### PDF
- 使用 `pypdf` 做基础正文提取。
- 优先读取 PDF 书签并回填 `work/toc_raw.md`、`work/chapter_index.json`。
- 逐页写出提取文本到 `work/extracted_text.md`。
- 检测页面内嵌图像对象并写入 `work/images_manifest.json`。
- 若目录或文本提取失败，会在 `00_readme.md` 和 `book_structure.json` 中显式降级说明。

该提取器是“可运行的基础层”，目标是把真实书籍输入自动落到中间文件中，而不是追求一次性达到最佳解析质量。


## 内置渲染层说明

`scripts/render_book.py` 会基于 `work/extracted_text.md`、`work/chapter_index.json`、`work/images_manifest.json` 生成：

- `output/chapters/chNN_章节精读.md` 的可读草稿
- `output/01_全书结构图.md`
- `output/02_核心论证.md`
- `output/03_关键概念.md`
- `output/04_易误读与限制.md`
- `output/05_复习问题.md`
- `output/06_图表索引.md`

该脚本的原则是：
- 能自动确定的内容直接写出
- 不确定的地方保留原文线索和明确提醒
- 画图时统一使用 Mermaid
- 不伪装成已经完成的高质量学术精读
